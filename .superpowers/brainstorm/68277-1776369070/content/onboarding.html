Key design decisions

  • 🔴 Own layout file — no SiteHeader/SiteFooter, full-screen focus
  • 🔴 Progress bar — thin red line at top, fills left-to-right across steps
  • 🔴 Language step first — sets tone, feels intentional
  • 🔴 Categories: pill multi-select — loaded live from GET /v1/categories
  • 🔴 Min 3 required — Continue button disabled until 3 are selected
  • 🔴 "Edit preferences" link on done screen — goes to /account
  • 🔴 Selections saved via topicsApi / user preferences endpoint (integration point clearly marked in code)
